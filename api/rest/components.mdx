---
title: Components
description: REST API endpoints for Meshery MeshModel components
---

MeshModel components represent infrastructure and application building blocks in Meshery. They define Kubernetes resources, service mesh configurations, and other cloud native primitives.

## List All Components

<CodeGroup>
```bash cURL
curl http://localhost:9081/api/meshmodels/components?page=0&pagesize=25
```

```javascript JavaScript
const response = await fetch(
  'http://localhost:9081/api/meshmodels/components?page=0&pagesize=25'
);
const data = await response.json();
```
</CodeGroup>

**Method:** `GET`

**Endpoint:** `/api/meshmodels/components`

### Query Parameters

<ParamField query="page" type="integer" default="0">
  Page number for pagination
</ParamField>

<ParamField query="pagesize" type="string" default="25">
  Number of components per page. Use `all` to return all results.
</ParamField>

<ParamField query="search" type="string">
  Search term for component name
</ParamField>

<ParamField query="order" type="string">
  Field to order by
</ParamField>

<ParamField query="sort" type="string" default="asc">
  Sort direction: `asc` or `desc`
</ParamField>

<ParamField query="annotations" type="string">
  Filter by annotation components:
  - `true` - Only return annotation components
  - `false` - Exclude annotation components
  - Any other value - Return all components
</ParamField>

### Response

<ResponseField name="page" type="integer">
  Current page number
</ResponseField>

<ResponseField name="page_size" type="integer">
  Number of items per page
</ResponseField>

<ResponseField name="total_count" type="integer">
  Total number of components
</ResponseField>

<ResponseField name="components" type="array">
  Array of component objects
  <Expandable title="Component Object">
    <ResponseField name="id" type="string">
      Unique component identifier
    </ResponseField>
    <ResponseField name="component" type="object">
      Component definition
    </ResponseField>
    <ResponseField name="displayName" type="string">
      Human-readable component name
    </ResponseField>
    <ResponseField name="kind" type="string">
      Component kind (e.g., Deployment, Service, VirtualService)
    </ResponseField>
    <ResponseField name="apiVersion" type="string">
      Kubernetes API version
    </ResponseField>
    <ResponseField name="schema" type="object">
      JSON schema for the component
    </ResponseField>
    <ResponseField name="model" type="object">
      Parent model information
    </ResponseField>
  </Expandable>
</ResponseField>

**Example Response:**

```json
{
  "page": 0,
  "page_size": 25,
  "total_count": 5432,
  "components": [
    {
      "id": "component-uuid",
      "component": {
        "kind": "Deployment",
        "schema": "...",
        "version": "v1.0.0"
      },
      "displayName": "Deployment",
      "kind": "Deployment",
      "apiVersion": "apps/v1",
      "model": {
        "name": "kubernetes",
        "version": "v1.28.0",
        "category": "Orchestration & Management"
      }
    }
  ]
}
```

## Register Components

<CodeGroup>
```bash cURL
curl -X POST \
  -H "Content-Type: application/json" \
  -d '{
    "modelName": "my-model",
    "components": [
      {
        "kind": "MyComponent",
        "apiVersion": "v1",
        "schema": {...}
      }
    ]
  }' \
  http://localhost:9081/api/meshmodels/components
```
</CodeGroup>

**Method:** `POST`

**Endpoint:** `/api/meshmodels/components`

### Request Body

<ParamField body="modelName" type="string" required>
  Model name for the components
</ParamField>

<ParamField body="components" type="array" required>
  Array of component definitions to register
</ParamField>

### Response

**Status:** `200 OK`

Returns the registered components.

## Component Schema

Get the JSON schema for a specific component:

<CodeGroup>
```bash cURL
curl http://localhost:9081/api/schema/resource/{resourceName}
```
</CodeGroup>

**Method:** `GET`

**Endpoint:** `/api/schema/resource/{resourceName}`

### Path Parameters

<ParamField path="resourceName" type="string" required>
  Resource/component name
</ParamField>

### Response

Returns the JSON schema definition for the component.
