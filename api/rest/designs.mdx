---
title: Designs
description: REST API endpoints for managing Meshery designs (patterns)
---

Designs (also called patterns) are declarative definitions of infrastructure and application configurations in Meshery. They allow you to define, version, and deploy cloud native infrastructure as code.

## List Designs

<CodeGroup>
```bash cURL
curl -H "Authorization: Bearer $TOKEN" \
  "http://localhost:9081/api/pattern?page=0&pagesize=25"
```

```javascript JavaScript
const response = await fetch(
  'http://localhost:9081/api/pattern?page=0&pagesize=25',
  {
    headers: {
      'Authorization': `Bearer ${token}`
    }
  }
);
const designs = await response.json();
```
</CodeGroup>

**Method:** `GET`

**Endpoint:** `/api/pattern`

### Query Parameters

<ParamField query="page" type="integer" default="0">
  Page number for pagination
</ParamField>

<ParamField query="pagesize" type="integer" default="10">
  Number of designs per page
</ParamField>

<ParamField query="search" type="string">
  Search term for filtering designs by name
</ParamField>

<ParamField query="order" type="string">
  Field to order by
</ParamField>

### Response

<ResponseField name="page" type="integer">
  Current page number
</ResponseField>

<ResponseField name="page_size" type="integer">
  Number of items per page
</ResponseField>

<ResponseField name="total_count" type="integer">
  Total number of designs
</ResponseField>

<ResponseField name="patterns" type="array">
  Array of design objects
</ResponseField>

## Create Design

<CodeGroup>
```bash cURL
curl -X POST \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My Application",
    "pattern_data": {
      "name": "My Application",
      "design_file": {
        "name": "my-app",
        "version": "0.0.1",
        "services": {}
      }
    },
    "save": true
  }' \
  http://localhost:9081/api/pattern
```

```javascript JavaScript
const response = await fetch('http://localhost:9081/api/pattern', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'My Application',
    pattern_data: {
      name: 'My Application',
      design_file: {
        name: 'my-app',
        version: '0.0.1',
        services: {}
      }
    },
    save: true
  })
});
```
</CodeGroup>

**Method:** `POST`

**Endpoint:** `/api/pattern`

### Request Body

<ParamField body="name" type="string" required>
  Design name
</ParamField>

<ParamField body="pattern_data" type="object" required>
  Design data including the pattern file
</ParamField>

<ParamField body="save" type="boolean" default="true">
  Whether to save the design
</ParamField>

### Response

**Status:** `201 Created`

Returns the created design object.
